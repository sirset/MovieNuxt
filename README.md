# MovieNuxt

## Build Setup

```bash
# install dependencies
$ npm install

# serve with hot reload at localhost:3000
$ npm run dev


# build for production and launch server
$ npm run build
$ npm run start

# generate static project
$ npm run generate
```

```bash
#preguntas 
1.	¿En qué consiste el principio de responsabilidad única? ¿Cuál es su propósito?

Consiste en distribuir las funcionalidades de un software en módulos y clases, de tal manera que los módulos definidos y clases van a tener responsabilidad sobre las funcionalidades del sistema a su encargo, generando una mayor desfragmentación de las aplicaciones a realizar y facilitando la descentralización de esta. Ocasionando facilidades como la flexibilización, y el testeo.

2.	¿Qué características tiene, según su opinión, un “buen” código o código limpio?

Un código limpio es aquel que sigue los siguientes alineamientos:

	Tenga una jerarquía al momento de ser codificado.
	Posea comentario donde sea necesario.
	No haya redundancia en el código.
	Los nombres asociados a atributos y métodos deben ser coherente con lo que realiza.
```
